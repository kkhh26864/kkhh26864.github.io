腾讯大楼下排长队装OpenClaw？我在家10分钟搞定，无需虚拟机

腾讯大楼下的”技术摆摊”让我开始思考

腾讯大楼前，看到一个有趣的场景：官方工程师团队在广场”摆摊”，现场帮路人部署 OpenClaw。队伍排得老长，可见这个 AI 工具有多火。

因为官方推荐的 WSL2 虚拟化方案，在我的笔记本上各种报错——虚拟化冲突、权限问题、环境变量丢失…折腾了两天还是没搞定。

后来我换了个思路：能不能直接在 Windows 11 原生环境跑，不要任何虚拟机？

答案是：可以。而且只需要一条关键指令。

三个避坑提醒（省你半天时间）

开始之前，先记住这三点：

PowerShell 会坑你

别用 PowerShell 运行 npm 命令。它的执行策略会莫名其妙拦截脚本，报一堆看不懂的错。

解决方案：全程用管理员权限的 cmd。

Git 不是可选项

很多人以为 npm 能搞定一切，结果装到一半报错：找不到 Git。

OpenClaw 的依赖包需要 Git 客户端才能拉取，提前装好能避免 90% 的安装失败。

网络要通畅

GitHub、npm 镜像、OpenAI API、Telegram——这些服务都要能访问。否则就算装上了也用不了。

关键一步：用 –legacy-peer-deps 强制安装

如果你直接运行 npm install -g openclaw@latest，大概率会遇到依赖冲突报错。

这是因为 OpenClaw 的某些依赖包在 Windows 上有兼容性问题。npm 默认的严格检查会直接拒绝安装。

解决办法：加上 --legacy-peer-deps 参数，让 npm 跳过这些检查。

这条指令会强制安装，忽略那些会导致冲突的对等依赖。

总结：原生部署的优势

虽然 Windows 原生环境踩坑多，但跑通后确实比 WSL2 轻盈很多：

  • 启动快：不用等虚拟机加载
  • 资源占用少：省了一层虚拟化开销
  • 路径问题少：不用在 Windows 和 Linux 之间来回转换文件路径

当然，数据备份很重要。OpenClaw 权限很高，误操作可能影响系统文件。
